## 한눈에 보는 구조 (4계층)

```
① 수집(Ingestion) → ② 문제은행(Bank) → ③ 출제(Generation) → ④ 검수·출력(Review)
```

1. **① 수집** — PDF·사진을 올리면 비전 AI가 문항 단위로 읽어 구조화 → 경량 AI 자동 검수 → 강사 검수 → 문제은행에 적재.
2. **② 문제은행** — 문제들을 과목·유형·난이도·개념으로 분류해 Postgres DB에 저장(+의미검색용 임베딩).
3. **③ 출제** — 정형 입력을 받아, **은행 전체가 아니라 지정 카테고리·구조화에 일치하는 문제만**(① SQL 필터로 후보 좁힘 → ② 코멘트 있으면 의미검색 top-K) 참고(RAG)해 AI가 K개 예시로 새 문제 생성 → 경량 AI 자동 검수. (매칭 0개면 순수 생성)
4. **④ 검수·출력** — 강사가 검토하고 AI에 피드백해 고치는 반복 루프 → 승인 → 시험지 PDF 출력 → 승인분은 다시 은행에 적재(플라이휠).

## 확정된 핵심 결정

1. **출제 = AI 생성 우선 + 강사 피드백 루프.** 새 문제를 만드는 게 목적이므로 AI 생성을 1순위로. 단 정답 오류 위험 때문에 **강사 검토 → AI 재생성**하는 반복 개선이 필수.
2. **수집 = 풀(full) 인제스천.** 업로드 기출을 문항 단위로 구조화·태깅·정답키 검수해 은행에 적재. ("할거면 제대로")
3. **구축 순서 = 텍스트형(영어·국어) 먼저** → 도형·수식이 많은 수학·과학은 그다음 증분 추가. 전 과목이 목표인 건 동일, 순서만.
4. **슈퍼셋 설계.** 텍스트형부터 데이터 모델을 도형·수식까지 담을 수 있게 만들어두고 비워둠 → 나중에 수학을 넣을 때 **갈아엎지 않고 채워 넣기**.
5. **경량 AI 자동 검수 게이트.** 강사 검수 *앞에* 가벼운 모델로 전수 점검 — 수집은 *글자 깨짐·구조 결함*, 출제는 *논리 결함·정답 재풀이*. **모델 티어링**: 생성=센 모델, 검수=경량 모델.
6. **모델 = 역할별 티어링.** 인제스천 추출=Sonnet(난페이지 Opus), 생성=Sonnet(킬러 Opus), 검수=Haiku 전수 + Sonnet/Opus 심층(킬러·수학), 강사 최종. **인제스천은 API로 실행(내 Claude Code 구독 아님)** — Batch(−50%)+캐싱으로 저비용. 교차 판정자는 안 둠. → 아래 "모델 선정" 표.

## 구현 반영

(2026-07-02) 기획대로 P1(생성→검수→피드백→승인→출력) + **양식/레이아웃**(P1.5) + **모의고사 코어**(P2)를 구현·배포했다. 수집/RAG·도형수식은 아직.

### P1 — 생성·검수·피드백·승인·출력
- **생성**: 정형 입력 → Claude tool-use 구조화 출력. 일반=Sonnet 4.6, 킬러=Opus 4.8. 부분/거부/구조깨짐은 예외 아닌 반환값(D6).
- **자동 검수**: Haiku 4.5 전수 트리아지 — 정답 재풀이·논리 결함을 `QualityFlag`(severity·code)로. 검수 거부 시 '미검수' 폴백.
- **피드백 루프**: 강사 피드백 → 전 문항 재생성(`GenerationTurn` 라운드).
- **승인 게이트("오답 인쇄 0")**: 미해결 ERROR 있으면 승인 차단(422) → 강사 검토 후 override 통과. 승인분은 `BankQuestion` 적재(플라이휠).
- **출력**: `@react-pdf/renderer`. NotoSansKR + 폰트 미포함 기호 sanitize(화살표·✓✗·이모지 → ASCII, tofu 0).
- **보안**: `manageQuestionBank`(proxy edge), 전 쿼리 academyId 스코프, 라우트 보안 통합테스트(IRON RULE).
- 파일: `src/lib/ai/*`·`src/lib/questionBank/*`·`src/app/api/question-bank/*`·`src/app/(admin)/questions/*`.

### P1.5 — 양식(프리셋) + 인쇄 레이아웃
**"양식"을 2축으로 분리**한 게 핵심(양쪽 다 폭발 안 함):
- **① 내용 = 학원이 프리셋 저장**(`TestPreset`: 이름 + spec + 레이아웃). 한 클릭 재출제. 학원이 무한히 만들되 개발 0.
- **② 인쇄 레이아웃 = 우리가 소수 제공**(`TestLayout`: `BASIC` 기본형 / `VOCAB` 단어시험형 2단). 프리셋이 선택. 물리적 시험지 형태는 3~4개로 수렴.
- **형식(format)**: 객관식/주관식 강제 — 단어시험 = 주관식 + VOCAB.

### P2 코어 — 모의고사(다중 섹션)
- **다중 섹션 모델**: `MockSpec`(subject·gradeLevel·title·`sections[]`), `TestLayout.MOCK`, `TestDraftItem.section`(섹션 인덱스; 단일 시험지는 0). 스펙에 `sections`가 있으면 모의고사(`isMockSpec`).
- **섹션별 순차 생성** (Hobby 60초 안에서 대형 시험 처리하는 방식): `POST /mock`(빈 초안 생성) → 클라이언트(`MockBuilder`)가 섹션마다 `POST /drafts/[id]/section {sectionIndex}` 순차 호출(각 ≤10문항, 생성+검수 포함). 마지막 섹션이면 status→REVIEW. `persistItems`가 section·orderOffset로 전역 order 연속 유지.
- **PDF**: MOCK 레이아웃 = 섹션 헤더(`[1] 어법`) + 전역 연속 번호. 검수·승인은 단일과 동일(전 문항 대상).
- **UI**: `/questions/mock` + `MockBuilder`(영역 빌더 + 순차 생성 진행표시). 목록·상세는 `isMockSpec`으로 분기, 상세는 섹션 그룹핑. 모의고사는 전체 피드백 대신 영역별 재생성(피드백 route는 MOCK 차단).
- **후속(미구현)**: 수능식 2단 지문박스 · OMR 답안지 · 섹션별 "재생성" 버튼(section route는 재생성 지원, UI만 없음).

## 바로잡은 관점 (왜 이렇게)

- **별도 vector DB 불필요** — 이미 쓰는 Neon Postgres에 `pgvector` 확장만 켜면 됨. 신규 벤더·비용 0.
- **분류는 RAG가 아니라 그냥 컬럼+검색** — 과목/유형/난이도/킬러는 일반 DB 필터. 임베딩(벡터)은 "자동사/타동사"처럼 태그로 못 박는 개념을 의미로 찾을 때만.
- **"구체 주제가 안 들어올 수도"의 해법 = 3중 분류** — 통제어휘(정형 필드) + 자유 개념태그 + 본문 임베딩. 코멘트가 비면 정형 필드로, 구체적이면 임베딩이 살림.
- **인식은 일반 OCR이 아니라 비전 LLM이 본체** — OCR은 글자만 주고 구조(보기·지문·정답 구분)를 못 줌. 비전 AI가 구조까지 이해해 JSON으로 출력.

## 모델 선정 (역할별)

작업 난이도에 맞춰 모델을 나눠 쓴다(모델 티어링). 비싼 모델은 어려운 일에만, 싼 모델은 자주 도는 일에. 교차(타벤더) 판정자는 두지 않는다.

| 역할 | 모델 | per 1M (입력/출력) | 왜 |
|---|---|---|---|
| 인제스천 추출 · 기본 | Claude Sonnet 4.6 | $3 / $15 | 비전 + 구조화 출력 |
| 인제스천 · 수식·도형 난페이지 | Claude Opus 4.8 | $5 / $25 | 고해상도 비전, 정밀 |
| 임베딩(벡터) | **별도 모델** (Claude 아님) | — | Anthropic 임베딩 미제공 → 한국어 강한 다국어 임베딩 따로 |
| 생성 · 일반 문항 | Claude Sonnet 4.6 | $3 / $15 | 균형 |
| 생성 · 킬러/최고난도 | Claude Opus 4.8 | $5 / $25 | 논리 정밀성 최상 |
| 검수 · 전수 트리아지 | Claude Haiku 4.5 | $1 / $5 | 싸고 빠름 |
| 검수 · 심층 판정 (킬러·수학만) | Sonnet/Opus | — | 정답 재풀이·논리 |

- 전 모델 비전·구조화 출력·캐싱 지원, 컨텍스트 1M(Haiku만 200K).
- **인제스천은 API로 실행 — Claude Code 구독 아님.** (구독을 제품 백엔드로 쓰면 약관·계정 리스크 + 학원 데이터 격리 붕괴 + 비확장. 구독은 '만들기·검증'에만.)
- **비용 레버**: 대량 인제스천 = Batch(−50%), 추출 스키마·루브릭 = 캐싱(읽기 ~0.1×), 구조화 출력.

### 검수: AI냐 강사냐? → 'AI 다음에 강사'

강사는 어차피 최종 게이트. AI 자동 검수는 강사를 *대체*하는 게 아니라, 플래그 찍힌 초안을 줘서 **어디를 볼지** 알려준다.
- **비용(자릿수)**: 시험지(10문항·3라운드)당 전수 트리아지(Haiku) ~40원, 심층 판정 더해도 ~200원. (실측 필요)
- **가치**: 강사 시급 ~3만원이면 1분 ≈ 500원 → 자동 검수가 1분만 줄여도 비용 2.5배 회수. + 조용한 오답을 미리 잡음.
- **결정**: Haiku 전수 트리아지 **항상 유지**, 심층 판정은 **킬러·수학에만**, 강사 **최종 결재**. (동전 몇 닢으로 강사 시간을 사는 거라 끄는 게 손해.)

## 데이터 모델 스케치 (forward-compatible)

```
Question       문항: 과목·학년·유형·난이도·킬러여부·개념태그 + content(블록: text|math|figure)
               + 정답 + 출처(AI생성/기출/변형) + 임베딩 + 검수상태
QuestionAsset  도형 자산(이미지·좌표). 텍스트형 V1엔 비어 있음, 수학에서 채움
TestDraft      시험지 초안: 스펙(과목·문항수·난이도·코멘트) + 상태 + 작성자
GenerationTurn 피드백 루프 이력: 강사 피드백 ↔ AI 수정 (되돌리기·비용 추적)
QualityFlag    자동 검수 플래그: 단계·심각도·코드(글자깨짐/정답없음/복수정답/난이도불일치…)
SourceDocument 업로드 원본(PDF·사진) + 파싱 상태
```

## 후속 작업 백로그 (구현 준비됨)

새 세션이 바로 착수 가능하게 규모·시작점·핵심을 적어둠. 착수 전 위 [이어받기 가이드](#-새-세션-이어받기-가이드) 필독.

### 1. 모의고사 정교화 (중간 규모) — P2a 후속
현재 MOCK은 "섹션 헤더 + 연속 번호"의 1단 시험지. 정교화 3가지:
- **수능식 2단 레이아웃**: `pdfDocument.ts`에 MOCK 2단 컬럼(특히 독해 지문 박스). 지금 구조에 컬럼 분할 추가.
- **OMR 답안지**: 새 variant(예 `?variant=omr`)로 문항번호 격자+마킹칸 페이지. `pdfDocument.ts`에 빌더 추가, PDF route에 variant 분기.
- **섹션별 재생성 버튼**: `QuestionDraftDetail`의 MOCK 상세 각 섹션에 "재생성" 버튼 → 기존 `POST /drafts/[id]/section` 재호출(백엔드는 이미 재생성 지원, deleteMany+재생성).
- 시작점: `src/lib/questionBank/pdfDocument.ts`, `src/components/questionBank/QuestionDraftDetail.tsx`, PDF route.

### 2. 텍스트형 수집/RAG (큰 규모) — P2b
**→ 상세 설계(착수 준비됨): [`문제출제_P2b_수집RAG_설계.md`](./문제출제_P2b_수집RAG_설계.md)** — 파이프라인·결정3개(임베딩벤더/비동기/검수)·데이터모델·단계·체크리스트.
업로드로 문제은행을 채우고, 생성 시 그것을 근거(RAG)로 쓰는 단계. 이게 "문제은행"의 본체.
- **수집**: 업로드(PDF·사진) → 비전 LLM(Sonnet, 난페이지 Opus) 구조화 → 자동검수(글자깨짐·구조) → 강사검수 → `BankQuestion` 적재. 새 `SourceDocument` 모델(원본·파싱상태).
- **RAG 그라운딩**: 생성 시 `BankQuestion`을 SQL 필터(과목·유형·난이도)로 후보 좁힘 → 코멘트 있으면 pgvector 의미검색 top-K → 예시로 프롬프트 주입. (매칭 0개면 지금처럼 순수 생성)
- **임베딩**: Anthropic 미제공 → 한국어 강한 벤더 선정(Voyage/OpenAI/Cohere). `BankQuestion.embedding`(pgvector) 채움. Neon에 pgvector 확장 켜기.
- **제약**: 업로드 파싱은 60초 초과 가능 → **비동기 잡 인프라 필요**(Vercel 함수 밖, 큐/워커). 대량은 Batch API(−50%)+캐싱.
- 시작점: 새 `src/lib/ai/ingest.ts` + `src/app/api/question-bank/ingest/`, `SourceDocument` 스키마, pgvector.

### 3. 도형·수식 (큰 규모) — P3
슈퍼셋 `content` 블록에 `math`(latex)·`figure`(assetId)는 **이미 정의됨** — 채우기만 하면 갈아엎기 없음.
- **수식**: 생성·저장은 LaTeX, 렌더는 KaTeX→이미지(또는 @react-pdf 수식). `pdfContent`/`pdfDocument`의 math 분기(현재 latex 원문 자리표시자) 구현.
- **도형**: `BankQuestionAsset`(Vercel Blob **private** — put는 반드시 access 없이, 프록시로 서빙) 채움. figure 블록 렌더.
- 시작점: `pdfContent.ts`/`pdfDocument.ts` math·figure 분기, `BankQuestionAsset`.

### 4. 운영·품질 (작음, 아무 때나)
- **Pro 전환**: `spec.ts` MAX_COUNT + generate·feedback route `maxDuration` 상향 → 대형 단일 시험 허용(모의고사는 이미 섹션 분할이라 무관).
- **비용 쿼터**: 학원별 월 생성 한도·경고.
- **검수 심층 판정**: 킬러·수학은 Haiku→Sonnet/Opus 에스컬레이션(현재 Haiku 전수만).
- **eval 확대**: `eval/`에 생성 구조 eval 추가(현재 검수 eval만). 골든셋 확장.
