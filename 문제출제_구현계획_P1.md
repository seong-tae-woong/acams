## 데이터 모델 (P0 · Prisma, forward-compatible, pgvector 제외)
```
enum QuestionSource   { AI_GENERATED  VARIANT  INGESTED }        // INGESTED는 P2
enum ReviewStatus     { DRAFT  PENDING  APPROVED  ARCHIVED }
enum TestDraftStatus  { GENERATING  REVIEW  APPROVED  ARCHIVED }
enum TurnRole         { TEACHER_FEEDBACK  AI_REVISION  AI_GENERATION }
enum FlagSeverity     { ERROR  WARNING }
enum FlagCode         { GARBLED_TEXT  MISSING_CHOICE  NO_CORRECT_ANSWER  MULTIPLE_CORRECT
                        ANSWER_MISMATCH  DIFFICULTY_MISMATCH  DUPLICATE  INCOMPLETE }

BankQuestion       { id, academyId, subject, gradeLevel, type, difficulty, isKiller,
                     conceptTags String[], content Json /*{blocks:[{type:'text'|'math'|'figure',…}]}*/,
                     answer Json, explanation Json?, source(QuestionSource), reviewStatus(ReviewStatus),
                     createdBy, createdAt, @@index([academyId, subject, gradeLevel, type, difficulty]) }
BankQuestionAsset  { id, questionId→BankQuestion, blobUrl, bbox Json, kind }   // P1엔 비어있음(슈퍼셋)
TestDraft          { id, academyId, spec Json /*과목·학년·유형·난이도·문항수·코멘트*/,
                     status(TestDraftStatus), createdBy, createdAt, @@index([academyId, createdBy]) }
TestDraftItem      { id, testDraftId→TestDraft, questionId→BankQuestion?, content Json, answer Json,
                     order Int, @@index([testDraftId]) }
GenerationTurn     { id, testDraftId→TestDraft, round Int, role(TurnRole), input Json, output Json?,
                     model String, tokensIn Int?, tokensOut Int?, createdAt, @@index([testDraftId, round]) }
QualityFlag        { id, testDraftItemId→TestDraftItem, stage String /*GENERATION*/, severity(FlagSeverity),
                     code(FlagCode), message String, model String?, resolved Boolean @default(false),
                     resolvedBy String?, createdAt, @@index([testDraftItemId, resolved]) }
```
- 마이그레이션은 **`prisma db push`**(배포 관례). `QuestionContent` 타입은 `src/lib/types/questionBank.ts` 단일 정의.
- P2 추가분(연기): `SourceDocument`, `BankQuestion.embedding vector`, pgvector 확장 — nullable 컬럼 추가라 재작성 없음.

## API 표면 (`src/app/api/question-bank/`)
| 라우트 | 메서드 | 설명 | 가드 |
|--------|--------|------|------|
| `/generate` | POST(stream) | 스펙→생성→자동검수→TestDraft 저장 | requireAuth + manageQuestionBank · ≤20 · academyId |
| `/drafts` · `/drafts/[id]` | GET | 초안 목록·상세(include items+flags+turns) | 동상 |
| `/drafts/[id]/feedback` | POST(stream) | 피드백 라운드 재생성 → GenerationTurn | 동상 |
| `/drafts/[id]/approve` | POST | status=APPROVED + BankQuestion 적재(플라이휠) | 동상 |
| `/drafts/[id]/pdf` | GET | @react-pdf 시험지+정답지 | 동상 |
- proxy: `TEACHER_PAGE_RULES`에 `['/questions','manageQuestionBank']`, `TEACHER_API_RULES`에 `['/api/question-bank','manageQuestionBank']` 추가.

## 데이터 흐름
```
[강사] 정형입력 → POST /generate (maxDuration=300, AI는 txn 밖)
  → Claude 생성(stream) + Haiku 자동검수 → 빠른 txn: TestDraft+Item[]+Turn(0)+Flag[]
  → 검수 UI(플래그 우선) → 피드백(재생성) → 승인 → BankQuestion 적재 → PDF
```

## AI 레이어 (`src/lib/ai/`)
- `client.ts` — Anthropic SDK 클라이언트(`@anthropic-ai/sdk`), `ANTHROPIC_API_KEY`.
- `models.ts` — 역할별 ID: 생성 `claude-sonnet-4-6`(킬러 `claude-opus-4-8`), 검수 `claude-haiku-4-5`.
- `schema.ts` — tool-use JSON 스키마(문항 구조), `QuestionContent`.
- `generate.ts` — 스펙→프롬프트(캐시 stable-first)→stream→파싱, 부분/refusal/JSON깨짐 처리(D6).
- `review.ts` — `reviewGeneratedQuestions(items)`→`QualityFlag[]`(코드별 + 정답 재풀이). P2에서 인제스천 게이트로 일반화.
- 에러는 기존 `logServerError`/`ErrorLog` 재사용. 토큰/비용은 GenerationTurn에 기록.

## NOT in scope (이번 사이클 제외)
- **인제스천**(PDF/사진 업로드·비전 LLM 파싱·검수 적재) → P2. 이유: 가장 비싸고 위험한 비전 파싱, 핵심 루프 검증 후.
- **RAG/pgvector/임베딩** → P2. 이유: P1은 생성만, 나중 nullable 컬럼 추가(재작성 없음).
- **수학·과학(도형·수식 LaTeX 렌더)** → P3. 이유: CV+KaTeX 파이프라인 별도.
- **잡 큐/폴링 인프라** → P2 배치 인제스천 때. 이유: P1 인터랙티브엔 조기.
- **교차(타벤더) 판정자** → 안 함(설계 결정).
- **CI 실제 LLM eval** → 안 함(flaky). 오프라인 골든셋으로.
