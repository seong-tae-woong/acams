# 학원로그 마케팅 페이지 디자인 시스템

> 대상: 공개 마케팅 라우트 `/intro` (신규 학원 원장 유치용 전환 자산)
> 미학: **Refined Minimal (Toss-clean)** — 흰색 우위, 넉넉한 여백, 굵고 친근한 한글 헤드라인, 둥근 카드, 절제된 강조색
> 원칙: 제품(`globals.css`) 토큰을 그대로 상속 → 리드가 데모에서 본 앱과 같은 색/질감 → 신뢰의 연속성

## 2. 타이포그래피

- **패밀리**: Pretendard 단일 (한글 클린 웹 표준). CDN 또는 self-host.
- **굵기**: Hero 800 / Heading 700·800 / Body 400·500
- **숫자**: `font-variant-numeric: tabular-nums` — 인강 수익·가격·KPI 등 모든 숫자에 적용 (정렬·신뢰감)
- **트래킹**: 큰 헤드라인은 `letter-spacing: -.02em` 내외로 타이트하게

| 단계 | 크기(desktop) | 굵기 |
|------|--------------|------|
| Hero | 48–52px | 800 |
| H2 (섹션) | 34–38px | 800 |
| H3 (카드) | 17–20px | 700 |
| Body | 16–17px | 400/500 |
| Small | 13–14px | 500 |

## 3. 간격·형태·모션

- **간격**: 8px 베이스. 섹션 상하 패딩 80–120px(desktop)로 넉넉하게.
- **radius**: 카드 16px · 버튼 10px · 배지/칩 pill(999px) — 제품 내부(카드 12 / btn 8)보다 약간 더 둥글게 해서 마케팅 톤을 부드럽게.
- **그림자**: 3단계(sm/md/lg). 히어로 제품 샷·인강 비주얼에 lg, 일반 카드 hover에 md.
- **모션**: 최소 + 스크롤 시 subtle fade/slide-up. 버튼 hover는 `translateY(-2px)` + 그림자 상승. 과한 애니메이션 금지.

---

## 4. 페이지 구조 & 메시지 위계

> 1차 메시지 = **통합/잔업 소멸** (teal·navy). 2차 강조 = **인강 수익** (보라 한정). 기능은 "없애주는 잔업" 중심.

1. **네비** — 학원로그 로고 + 상담 문의 CTA
2. **히어로** — "여러 프로그램·엑셀·카톡, 학원로그 하나로 끝냅니다" + 대시보드 제품 샷. 1차 CTA(데모 신청).
3. **공감(현재의 고통)** — "흩어진 도구 → 하나" 변환 비주얼: 좌측 회색·기울어진 도구 칩(종이 출결·엑셀 정산·카톡 공지·계산기) → 화살표 → 우측 정돈된 teal "통합 대시보드" 카드(BrandMark + ✓행 3개). AS-IS는 무채색, TO-BE는 teal/navy로 대비.
4. **인강 수익 섹션 (보라 한정)** — "학생은 스스로, 원장은 관리만" + 큰 tabular 숫자 + 강의 진도 리스트.
5. **기능 7종** — 각각 "없애주는 잔업" 한 줄: 통합 관리 / 성적 리포트 / 알림 자동화 / 학원별 공개페이지 / 출결 자동화 / 학원비 청구·결제 / 수업·보강 일정.
6. **모바일·학부모** — 폰 목업(출결·성적·인강 진도).
7. **최종 CTA** — 카카오톡 데모 신청.

## 7. 브랜드 마크 (학원로그 아이콘)

> 단일 소스(기하학 SVG)에서 모든 아이콘을 파생. 폰트에 의존하지 않아 1px~512px 어디서나 선명.

### 7.1 마크 구성
- **글자**: `HL`(Hakwon-Log)을 직선 `<rect>`로 그린 기하학 레터폼. H = 좌측 기둥 + 우측 기둥 + 가로줄, L = 세로 기둥 + 받침. viewBox 100×100, 글자 영역 x21–79 · y28–72(중앙 정렬).
- **바탕**: teal 그라데이션 사각형. 좌상(`#12B886`) → 우하(`#4FC3A1`) 대각 linearGradient.
- **글자색**: 항상 흰색(`#fff`).

### 7.2 변형(variant)
| 용도 | 바탕 | 모서리 | 글자 스케일 |
|------|------|--------|------------|
| 화면/파비콘 (≥28px) | teal 그라데이션 | 둥근 사각 `rx=24%` (투명 모서리) | 1.0 |
| 소형 (<28px) | **단색** `#12B886` | 둥근 사각 | 1.0 |
| Apple touch / PWA maskable | teal 그라데이션 | **풀블리드**(직각, 투명 없음) | 0.86–0.9 (안전영역 축소) |

- **단색 폴백**: 28px 미만에서는 그라데이션 디테일이 뭉개지므로 solid `#12B886`로 자동 전환(`BrandMark variant="auto"`).
- **maskable 규칙**: OS가 원/스쿼클 등으로 마스킹하므로 바탕을 풀블리드로 채우고 HL을 중앙 안전영역(약 80%) 안에 둔다. → 글자 스케일을 줄여 가장자리 잘림 방지.

### 7.3 워드마크 연계
- 텍스트 로고 `학원로그`의 **"로그"** 색은 마크 앵커색과 통일: `#12B886`. ("학원"은 배경 대비용 `hakwonColor` prop.)
- 마크 + 워드마크 병기 시 마크를 좌측, 워드마크를 우측에 baseline 정렬.

### 7.4 클리어 스페이스 & 최소 크기
- 마크 사방 여백 ≥ 마크 한 변의 **20%**.
- 최소 표시 크기 16px(파비콘). 그 이하로 축소 금지.

### 7.5 금지(Don't)
- 비례 왜곡·회전·기울임 금지.
- 그라데이션 색을 임의 변경 금지(teal 고정). 보라(인강 테마)로 칠하지 말 것.
- HL 글자색을 흰색 외 색으로 바꾸지 말 것.
- 마크 위에 그림자·외곽선·테두리 추가 금지.

### 7.6 구현 & 에셋 재생성
- **온스크린 컴포넌트**: `src/components/shared/BrandMark.tsx` (`size`, `variant='gradient'|'solid'|'auto'`, `title` props). 서버 컴포넌트 호환(고정 gradient id).
- **텍스트 로고**: `src/components/shared/Wordmark.tsx`.
- **래스터 에셋 생성**: `node scripts/gen-brand-icons.mjs` (sharp). 단일 SVG 소스에서 아래를 재생성:
  - `src/app/icon.png` (256, 둥근, 파비콘) · `src/app/apple-icon.png` (180, 풀블리드)
  - `public/icon-192.png` · `public/icon-512.png` (풀블리드, manifest `purpose: "any maskable"`)
- 색/형태 기준을 바꾸면 **반드시** 스크립트를 다시 실행해 4개 PNG를 일괄 갱신(소스 1곳 → 에셋 동기화).
